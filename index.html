제안 포인트

본 프로젝트는 단순 디자인이 아닌, 운영 로직 및 기능 구조 중심의 서비스 설계가 핵심입니다. 다음과 같은 접근으로 개발팀이 바로 구현할 수 있는 산출물을 제공합니다:

  • ERD/API 기반 역설계 — 기존 DB 구조와 API 명세서를 분석하여 데이터 흐름에 맞는 화면 설계. 개발 단계에서 구조 변경 최소화

  • 예외 케이스 선제 설계 — 부분 취소, 정산 오류, 결제 실패, 딜 무산 등 실서비스 운영 시 발생하는 예외 시나리오를 기획 단계에서 정의
  • TOSS 스타일 UX — 복잡한 커머스 관리 기능을 사용자 친화적인 UI/UX로 풀어내어, 판매자/관리자의 학습 비용 최소화

PRD (Product Requirements Document)

판매자/관리자 페이지의 기능 요구사항을 정의하고 범위를 명확히 합니다.

1. 프로젝트 배경 및 목적

공동구매 플랫폼 서비스를 개발 중이며, 소비자 앱은 결제/배송 일부를 제외하고 주요 기능이 구현된 상태입니다. 본 프로젝트는 실서비스 운영이 가능한 수준의 판매자 페이지와 관리자 페이지를 기획·설계하여, 개발 단계로 바로 이어질 수 있는 산출물을 확보하는 것이 목적입니다.

특히 단순 쇼핑몰이 아닌 참여자 수 기반 실시간 할인 구조 및 공동구매 로직을 포함한 플랫폼이므로, 이에 맞는 특화 설계가 필요합니다.

2. 사용자 정의

사용자 유형설명주요 기능
판매자상품을 등록하고 공동구매 딜을 운영하는 사업자상품 관리, 딜 생성, 주문/배송/환불 관리, 정산 확인, 고객 응대
관리자플랫폼 전체를 운영·관리하는 내부 담당자판매자/사용자 관리, 상품/딜 승인, 주문/결제/정산 관리, 통계, 커뮤니티 관리
소비자공동구매에 참여하는 일반 사용자 (기존 앱 — 범위 외)상품 탐색, 공동구매 참여, 결제, 배송 조회

3. 기능 요구사항

3.1 판매자 페이지

  • [필수] 대시보드 — 매출 현황, 주문 현황, 공동구매 진행 현황 요약
  • [필수] 상품 등록/수정/삭제 — 카테고리, 이미지, 가격, 재고, 옵션 관리
  • [필수] 공동구매 딜 생성 — 참여자 수 기반 할인율 설정, 딜 기간, 최소/최대 참여자 수, 딜 상태 관리
  • [필수] 주문 관리 — 주문 목록, 상태 변경(확인→배송준비→배송중→배송완료), 송장 번호 등록
  • [필수] 배송 관리 — 배송 추적, 일괄 송장 등록
  • [필수] 취소/환불 처리 — 전체 취소, 부분 취소, 환불 사유 입력, 환불 상태 관리
  • [필수] 정산 관리 — 정산 내역 조회, 정산 주기 확인, 수익 통계
  • [필수] 고객 문의 — 문의 목록, 답변 작성, 상태 관리(대기/답변완료)
  • [선택] 리뷰 관리 — 리뷰 목록, 답글 작성
  • [선택] 판매자 프로필/설정 — 사업자 정보, 배송 정책, 알림 설정

3.2 관리자 페이지

  • [필수] 대시보드 — 전체 매출, 가입자 수, 활성 딜 수, 주요 KPI 차트
  • [필수] 판매자 관리 — 판매자 목록, 상세 조회, 승인/거절, 상태 변경
  • [필수] 사용자(소비자) 관리 — 회원 목록, 상세 조회, 상태 변경
  • [필수] 상품 관리 — 상품 승인/반려, 카테고리 관리
  • [필수] 딜(공동구매) 관리 — 딜 승인/반려, 딜 현황 모니터링, 강제 종료
  • [필수] 주문 관리 — 전체 주문 조회, 이슈 건 처리
  • [필수] 결제 관리 — 결제 내역, 결제 실패 처리
  • [필수] 환불 관리 — 환불 요청 처리, 부분 환불 승인
  • [필수] 정산 관리 — 정산 내역, 정산 승인, 정산 오류 처리
  • [필수] 통계 — 매출, 사용자, 상품, 공동구매 참여율 통계
  • [선택] 커뮤니티 관리 — 제안/댓글 관리, 신고 처리
  • [선택] 운영 설정 — 공지사항, 배너, 약관 관리

3.3 공통 로직 설계

  • [필수] 공동구매 딜 상태 흐름 — 생성→모집중→목표달성→종료→정산 상태 전이
  • [필수] 결제 흐름 — 결제 요청→결제 완료→부분취소→전체취소 흐름
  • [필수] 정산 흐름 — 주문완료→정산대기→정산승인→정산완료 흐름
  • [필수] 예외 처리 — 딜 무산 시 전체 환불, 부분 취소 시 할인율 재계산, 정산 오류 수동 처리
